## Key Directories Explained

### `/convex`
Backend serverless functions and database schema. Each file contains:
- Query functions (read operations)
- Mutation functions (write operations)
- Type-safe validators
- Row-level security

### `/src/pages`
Full-page React components, one per route:
- Authenticated pages (Dashboard, Jobs, etc.)
- Public pages (Landing)
- Role-specific pages (CoachDashboard)

### `/src/components`
Reusable UI components used across multiple pages:
- Cards for displaying data
- Navigation components
- Messaging components

### `/src/hooks`
Custom React hooks for common patterns:
- `useCurrentUser` - Get current authenticated user
- More hooks can be added as needed

### `/src/lib`
Utility functions and helpers:
- `utils.ts` - Date formatting, ATS scoring, currency formatting, etc.

### `/src/types`
TypeScript type definitions:
- Interfaces matching Convex schema
- Type safety across frontend and backend

## Database Schema Overview

### 13 Tables with Relationships

1. **users** → foundational table
   - Referenced by: starStories, resumes, jobs, applications, coaches, sessions, reviews, payments, subscriptions, messages

2. **starStories** → referenced by resumes

3. **resumes** → referenced by verificationTasks, applications

4. **jobs** → referenced by applications

5. **coaches** → referenced by sessions, bids, reviews
   - References: users

6. **sessions** → referenced by reviews, payments
   - References: users, coaches

7. **verificationTasks** → referenced by bids, reviews, payments
   - References: resumes, users

8. **bids** → references: verificationTasks, coaches

9. **reviews** → references: users, coaches, verificationTasks, sessions

10. **payments** → references: users, verificationTasks, sessions

11. **subscriptions** → references: users

12. **applications** → references: users, jobs, resumes

13. **messages** → references: users (sender), users (receiver)
